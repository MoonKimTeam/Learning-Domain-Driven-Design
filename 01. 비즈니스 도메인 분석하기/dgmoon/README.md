# ch.01 비즈니스 도메인 분석하기

- 효과적인 솔루션을 설계하고 구축하려면 그것의 바탕이 되는 문제를 이해해야 한다.

## 비즈니스 도메인이란

- 기업의 주요 활동 영역
- 회사가 고객에게 제공하는 서비스
- ex) 우버: 차량 공유 서비스, 음식 배달 및 자전거 공유 서비스

## 하위 도메인이란?

- 비즈니스 활동의 세분화된 영역
- 유형
    - 핵심 하위 도메인: 회사가 경쟁업체와 다르게 수행하고 있는 것
        - 핵심 하위 도메인은 자연스럽게 복잡해짐
        - 회사의 경쟁 우위는 다양한 원천에서 나옴
    - 일반 하위 도메인: 모든 회사가 같은 방식으로 수행하는 비즈니스 활동
    - 지원 하위 도메인: 회삭의 비즈니스를 지원하는 활동
- 비교
    - 핵심 하위 도메인만이 회사에 경쟁 우위를 제공함
    - 일반 하위 도메인은 경쟁 우위의 원천이 될 수 없음
    - 지원 하위 도메인은 진입장벽이 낮고 경쟁 우위도 제공할 수 없음
- 복잡성
    - 핵심 하위 도메인은 복잡하다
- 변동성
    - 핵심 하위 도메인은 자주 변경될 수 있고, 지원 하위 도메인은 그렇지 않음
- 전략
    - 핵심 하위 도메인은 사내에서 구현되어야 하며, 하청하는 것은 현명하지 못함
- 하위 도메인 식별하기
    - 회사와 기타 조직 단위서부터 출발하지만, 비즈니스 기능의 복잡한 내용에 숨겨진 정보를 놓쳐선 안 됨
    - 기술적인 관점에서 하위 도메인은 상호 연관되고 응집된 유스케이스의 집합과 유사
    - 관련되지 않은 비즈니스 기능을 식별하고, 그자체로 인정하며, 작업중인 시스템과 관련된 비즈니스에 집중하는 것이 중요

## 도메인 분석 예제

- Gigmaster: 티켓 판매 및 유통회사
    - 비즈니스 도메인: 티켓 판매
    - 핵심 하위 도메인: 추천 엔진, 데이터 익명화, 모바일 앱
    - 일반 하위 도메인: 암호화, 회계, 정산, 인증 및 권한 부여
    - 지원 하위 도메인: 음악 스트리밍 서비스와 연동, 소셜 네트워크와 연동, 참석 모듈 연동
    - 설계 전략
        - 추천 엔진, 데이터 익명화, 모바일 앱은 가장 진보된 엔지니어링 도구와 기술을 사용하여 사내에서 구현돼야 함
        - 데이터 암호화, 회계 정산, 인증에는 상용 또는 오픈 소스 솔루션을 사용해야 함
        - 스트림 서비스, 소셜 네트워크와 연동과 참석한 공연을 위한 모듈 개발은 하청할 수 있음

## 도메인 전문가는 어떤 사람인가?

- 우리가 모델링하고 코드로 구현할 비즈니스의 모든 복잡성을 알고 있는 주제 전문가
- 일반적으로 요구 사항을 제시하는 사람 또는 소프트웨어 최종 사용자
