# ch.07 시간 차원의 모델링

## 이벤트 소싱

- 이벤트 소싱 패턴은 데이터 모델에 시간 차원을 도입함

### 검색

- 이벤트 소싱을 사용하면 과거 정보를 쉽게 프로젝션할 수 있음

### 분석

- 실제로 필요한 기능을 구현하려면 프로젝션된 모델을 데이터베이스에 유지해야 함

### 원천 데이터

- 이벤트 소싱 패턴이 작동하려면 객체 상태에 대한 모든 변경 사항이 이벤트로 표현되고 저장되어야 함
- 이벤트를 저장하는 데 사용되는 데이터베이스를 지칭하는 이름이 이벤트 스토어

### 이벤트 스토어

- 이벤트 스토어는 추가만 가능한 저장소이므로 이벤트를 수정하거나 삭제할 수 없음

## 이벤트 소싱 도메인 모델

- 이벤트 소싱 도메인 모델은 애그리게이트의 수명주기를 모델링하기 위해 독점적으로 도메인 이벤트를 사용함

### 장점

- 시간 여행
- 심오한 통찰력
- 감사 로그
- 고급 낙관적 동시성 제어

### 단점

- 학습 곡선
- 모델의 진화
- 아키텍처 복잡성
